Channels

Channels are used to provide information to the OPC Server about the physical connection to a hardware device (PLCs). For the Modbus/TCP server, a channel is comprised of the target device’s IP Address, IP Port, and connection timeout value. A unique channel must be created for each IP Address and IP Port pair.

Adding a Channel

To add a channel, select the OPC, Channels… menu item or the Channels icon
on the server toolbar. A Channel Configuration dialog will appear:

Select the [Add…] button to add a new channel. The Channel Properties dialog will appear:

  1. Enter the IP Address of the target device.
  2. If a non-standard port is used by the target device, enter it in the IP Port field. The default for Modbus/TCP is 502.
  3. Enter the connection timeout value in milliseconds.
  4. Click [OK] to create the new channel and close the dialog, or click [Cancel] to close the dialog without creating a new channel.

Changing a Channel

To edit a channel configuration, select the OPC, Channels… menu item or the Channels icon
on the server toolbar. A Channel Configuration dialog will appear:

Select a channel from the Installed Channels list box and click the [Edit…] button. The Channel Properties dialog will appear:

  1. Make the appropriate changes.
  2. Click [OK] to create the new channel and close the dialog, or click [Cancel] to close the dialog without creating a new channel.

Deleting a Channel

To delete a channel configuration, select the OPC, Channels… menu item or the Channels icon
on the server toolbar. A Channel Configuration dialog will appear:

Select a channel in the list box and click the [Delete] button. The Channel will be removed from the Installed Channels list box.

Devices

OPC Server Devices are used to refer to hardware devices (PLC or other process hardware) that are connected to the OPC Server. For the Modbus/TCP OPC Server, devices refer to Modicon or other 3rd party Modbus/TCP compatible hardware. A Device contains information such as the PLC’s Node Address, transaction timeout, connection type, etc.

Adding a Device

Follow the steps below to add a hardware Device:

  1. Click on the Add Device
    toolbar button and the Device Properties dialog will appear:

  1. Enter the following information:

Device Name

Enter a name for the target device (process hardware). Device names must be unique on the server they were created on. Examples: MyDevice, Quantum_1, Packaging_Line

Channel

Dropdown list containing all the channels that are configured on the server. Select the appropriate channel for this device.

Unit ID

Modbus/TCP Unit ID. Default is 0, range is 0…254. The Unit ID is intended for use with bridging devices. Most Modbus/TCP slave hardware will ignore this value, but some hardware requires a valid Unit ID.

Transaction Timeout

Transaction timeout value for the target device in milliseconds.

Address Type

Select from the following choices:

  • Modicon Style PLC Addressing (x, 1xxxx, 3xxxx, 4xxxx)
  • Absolute Zero-based Addressing
  • Generic One-based Addressing

Force Discretes

The Modbus/TCP specification provides two functions for forcing discretes:

  • FNC 5 – Force Single Coil
  • FNC 15 – Force Multiple Coils

Hardware support for one or both these functions is manufacturer dependent. Select the radio button that is appropriate to your hardware’s functional support. If your hardware supports both, check either radio button; effect on performance is negligible.

Float Word Swap

For floating point variables only. The word order in which floating point data is stored and transferred via Modbus/TCP varies across manufacturers. If this box is unchecked, data is handled using non-IEEE word order. If this box is checked, data is handled using IEEE word order.

Long Word Swap

For long integer variables only. The word order in which long integer data is stored and transferred via Modbus/TCP varies across manufacturers. If this box is unchecked, data is handled using non-IEEE word order. If this box is checked, data is handled using IEEE word order.

Simulate I/O

Simulates activity on the device. Useful during development or when the target device is not present. Uncheck this when the target device is connected and actual data is required.

NOTE: When simulate I/O is checked, simulated values are generated for INPUT_REGISTERS and DISCRETE_INPUTS only; OUTPUT_REGISTERS and DISCRETE_OUTPUTS are not effected. Since input registers are read-only, you will not be able to write values to simulated registers.

  1. Click on the [OK] button to save changes, or click on the [Cancel] button to exit without saving changes.

  2. Right mouse click on the Device icon in the tree view pane and select Remove from the popup menu.- or -Click on the target Device in the tree view pane and then click on the Delete
    toolbar button.

Tag Groups

Tag groups provide a logical way to group tags together. Tag groups allow clients that support tag browsing to view tags hierarchically.

Tag groups should not be confused with OPC Client Groups. For a description of OPC Groups see the OPC 2.0 DA Custom Specification.

Adding a Tag Group

Follow the steps below to add a Tag Group :

  1. Click on the Add Group
    toolbar button and the Group Properties dialog will appear:

  1. Enter the following information:

· Name: Enter a name for the Tag Group.

  1. Click on the [OK] button to add the Tag Group. Click on the [Cancel] Button to exit without adding the Tag Group.

Changing a Tag Group

Follow the steps below to change a Tag Group :

  1. Right mouse click on the Tag Group
    icon in the tree view pane, select Properties… from the popup menu, and the Tag Group’s property dialog will appear:

  1. Make changes.
  2. Click on the [OK] button to save changes, or click on the [Cancel] button to exit without saving changes.

After making changes, it is not necessary to reboot the server.

Deleting a Tag Group

Follow the steps below to remove a Tag Group :

  1. All Tags that appear below the target device must be removed prior to removing the Tag Group.
  2. Right mouse click on the Tag Group icon in the tree view pane and select Remove from the popup menu.- or -Click on the target Tag Group in the tree view pane and then click on the Delete toolbar button .

Tags

An OPC Tag is an item on the OPC Server that represents a data point in the target device (process hardware).

Descriptive names can be used instead of PLC style numeric registers. For instance you could create a tag named Pressure which corresponds to register 30001.

Adding a Tag

Follow the steps below to add a Tag :

  1. Click on the Add Tag
    toolbar button and the Tag Properties dialog will appear:

  1. Enter the following information:
  • Name: Enter a name for the Tag.
  • Description: Optionally enter a description for the Tag.
  • Type: Select the Tag’s data type from the pulldown list.
  • Location: Enter the hardware register number or general reference file address. For tags that refer to general reference files, use this field to specify the general reference address. Format is ,,. Currently, the field must be set to 6.
  • Data Type: This is how the data will be represented in the OPC Server.
  • Number of Bytes: For VT_BSTR data type only. Specify the maximum string length here. Largest string allowable is 200 bytes.
  • BCD Format: Check this box if data is stored in the process hardware in BCD format. Valid only for VT_I2 and VT_I4 types.
  • Scaling and Alarming: Check this box to enable scaling and alarming. For non-discrete types only.
  • Simulation Signal: Applies only when the Device is set to Simulate. Specifies how the simulation values are varied.
  1. Click on the [OK] button to add the Tag Group. Click on the [Cancel] Button to exit without adding the Tag Group.

Changing a Tag

Follow the steps below to change a Tag :

  1. Right mouse click on the Tag icon in the list view pane, select Properties… from the popup menu, and the Tag Property dialog will appear:

  1. Make changes.
  2. Click on the [OK] button to save changes, or click on the [Cancel] button to exit without saving changes.

After making changes, it is not necessary to reboot the server.

Run Modes

The server can be configured to run in the following modes:

Unattended Mode

Unattended mode is set during DCOM Configuration.

Unattended mode means that in addition to normal server operation where a user is logged onto the OPC Server PC, the server can be launched from a remote OPC Client regardless of whether someone is logged onto the OPC Server PC.

When set to run in unattended mode and the OPC Server is launched from an OPC Client, the OPC Server’s user interface will not be shown. The server will only show up as a process in the Task Manager. Configuration changes cannot be made while in this mode.

When set to run in unattended mode and the OPC Server is launched manually, the OPC Server’s user interface will be present and configuration changes can easily be made.

To run the OPC Server in unattended mode, the OPC Server’s DCOM configuration must be properly set.

Refer to this document for setting up the server to run in unattended mode:DCOM and Windows Firewall Configuration for Automated Solutions OPC Servers

System Tray Mode

System Tray mode is set via the OPC Server’s View, Run popup menu in the System Tray.

The System Tray is the area located on the end of the Task Bar and contains icons of background processes. The system tray is shown below:

To activate the server’s user interface from the system tray, double click on the server icon.

Note: when the ‘Run in System Tray’ option is selected, you must use File|Exit to shut the server down. Clicking on the close button [X] will minimize the server to the system tray, not close the server.

Form the system tray you can also right-mouse click on the server icon, the select Exit from the popup menu to shutdown the server.

Standard Executable Mode

Standard Executable mode is set via the OPC Server’s View, Run popup menu in the System Tray.

When run as a standard executable, the standard OPC Server user interface is shown. When the server is minimized, it will minimize to the Windows Task Bar.
