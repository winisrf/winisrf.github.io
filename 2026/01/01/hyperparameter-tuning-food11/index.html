Hyperparameter Tuning Analysis for Food-11 Image Classification

Posted on 2026-01-01 Edited on 2026-07-03 In Tuning

Abstract. This report investigates the impact of several important hyperparameters and regularization methods on the image classification performance of deep neural networks. Experiments were conducted on the Food-11 dataset using the EfficientNet-B0 architecture. The study consists of four parts: learning rate comparison, learning rate schedule comparison, weight decay comparison, and mixup data augmentation. The results show that an initial learning rate of 0.025 provides the best balance between convergence speed and stability, constant learning rate outperforms cosine annealing in validation performance under the current setting, stronger weight decay improves generalization, and mixup provides a slight improvement in validation and test performance at the cost of lower training accuracy.

Data Preprocessing

The Food-11 dataset contains a total of 16,643 images, with 9,866 images in the training set, 3,430 images in the validation set, and 3,347 images in the test set. As per the assignment requirements, all images were scaled to 100×100. The code for this preprocessing is in Code 1 and Code 2.

Learning Rate

Introduction

This section tested three initial learning rates: 0.1, 0.025, and 0.001. All experiments used the SGD optimization algorithm with momentum set to 0.9, a batch size of 128, and 15 epochs per group.

Results

The results are summarized in Table 2.

Table 2: Performance under different initial learning rates.

Learning Rate Train Loss Train Acc. Val Loss Val Acc.
0.1 1.5989 44.46% 1.5720 45.36%
0.025 1.2397 57.75% 1.2853 56.21%
0.001 1.6722 42.05% 1.7323 41.02%

Analysis

The experimental results show that the learning rate of 0.025 significantly outperformed the other two groups. With a learning rate of 0.025, the model achieved the lowest training loss (1.2397) and the highest training accuracy (57.75%), as well as the lowest validation loss (1.2853) and the highest validation accuracy (56.21%). In contrast, with a learning rate of 0.1, the training loss was 1.5989 and the training accuracy was 44.46%, while the validation loss was 1.5720 and the validation accuracy was 45.36%. However, with a learning rate of 0.001, all metrics were the worst, with a training loss of 1.6722, a validation loss of 1.7323, and a validation accuracy of only 41.02%.

This demonstrates that when the learning rate is too large, the parameters are updated more rapidly at each step, making it more prone to oscillations during optimization and hindering the model from converging. In this experiment, the learning rate of 0.1 significantly lagged behind 0.025, indicating that this setting was too large for the current task and number of training epochs, affecting the model’s convergence quality. When the learning rate is too small, the parameter update pace is too slow, although the model is still learning, it often fails to converge sufficiently within a limited number of epochs. The learning rate of 0.001 showed the worst performance in both training and validation, as its learning process was too slow to learn enough effective features within the current training budget.

In summary, a learning rate of 0.025 achieved the best balance between convergence speed and training stability. On the one hand, it is not as aggressively updated as 0.1, on the other hand, it drives model optimization faster than 0.001, achieving optimal results on both the training and validation sets. Based on this experimental result, 0.025 can be selected as the initial learning rate for subsequent experiments.

Learning Rate Schedule

Introduction

After determining the optimal initial learning rate, the next step is to compare constant learning rate and cosine annealing learning rate scheduling, training each for 300 epochs.

Cosine annealing gradually decays the learning rate from its initial value to 0 according to a cosine curve, with the following formula:

ηt=ηmin⁡i+12(ηmax⁡i−ηmin⁡i)(1+cos⁡(TcurTiπ))\eta_t = \eta_{\min}^{i} + \frac{1}{2}\left(\eta_{\max}^{i} - \eta_{\min}^{i}\right)\left(1 + \cos\left(\frac{T_{\mathrm{cur}}}{T_i}\pi\right)\right) ηt​=ηmini​+21​(ηmaxi​−ηmini​)(1+cos(Ti​Tcur​​π))

Here, iii represents the run number, ηmin⁡i\eta_{\min}^{i}ηmini​ and ηmax⁡i\eta_{\max}^{i}ηmaxi​ represent the minimum and maximum values of the learning rate respectively, defining the range of the learning rate. TcurT_{\mathrm{cur}}Tcur​ indicates the number of epochs executed so far, TiT_iTi​ represents the total number of epochs in the ithi^{\text{th}}ith run.

Cosine annealing changes the originally fixed learning rate to one that gradually decreases with each epoch, and then observes whether the model performs better on the validation set. The learning rate changes smoothly from large to small.

Results

Table 3 reports the final results, and Figure 1 shows the corresponding training curves.

As shown in the figure, both methods continuously optimize on the training set, but cosine annealing achieves a more complete fit in the later stages. In the left graph, the green solid line (cosine annealing training loss) decreases more overall, while in the right graph, the green solid line (cosine annealing training accuracy) eventually surpasses the blue solid line of constant LR. Cosine annealing does indeed improve the model’s learning on the training set.

In the right graph, both validation accuracy curves tend to plateau around 0.70, but the validation accuracy of constant LR is slightly higher overall than that of cosine annealing; in the left graph, the validation loss of constant LR is also lower overall than that of cosine annealing. Although cosine annealing makes the training set metrics look better, it doesn’t lead to better generalization.

The graph exhibits some instability, with the validation curve showing obvious peaks or sudden drops, especially the validation set curve in the loss graph showing significant fluctuations. The validation process may be sensitive to the model parameters of certain epochs, but in the long run, this does not affect the overall conclusion.

Schedule Train Loss Train Acc Val Loss Val Acc
Constant LR 0.0327 98.92% 1.4974 73.06%
Cosine Annealing 0.0160 99.47% 1.7733 71.55%

Analysis

Based on the current experimental results, using Constant LR the model achieved a training loss of 0.0327, training accuracy of 98.92%, and a validation loss of 1.4974 with a validation accuracy of 73.06%. Using Cosine Annealing, the model achieved a training loss of 0.0160, training accuracy of 99.47%, and a validation loss of 1.7733 with a validation accuracy of 71.55%. Cosine annealing performed better on the training set, but its performance on the validation set was inferior to that of constant learning rate. Cosine annealing allows the model to fit the training set more fully, but this stronger fit does not translate into better generalization ability.

Cosine annealing gradually decreases the learning rate as training progresses, with increasingly smaller update steps in the later stages. This typically helps the model optimize the training objective more finely. However, in this experiment, this resulted in an overfitting tendency within the training set. A constant learning rate maintains a relatively fixed update amplitude, preventing parameters from prematurely entering a fine-fit state and exhibiting better fitting performance under the current conditions.

Weight decay directly reduces parameter values during optimization, thereby suppressing excessively large weights and reducing the risk of overfitting. This section uses the optimal learning rate obtained in Section 3 and cosine annealing from Section 4 to test two weight decay coefficients: 5×10−45 \times 10^{-4}5×10−4 and 1×10−41 \times 10^{-4}1×10−4 with 300 training epochs for each.

The red dashed line for WD = 0.0001 stabilizes at approximately 0.71 in the later stages. The larger weight decay (0.0005) results in higher validation accuracy, while the smaller weight decay (0.0001) although almost fully learning the training set, performs significantly worse. Meanwhile, the validation loss in the left graph suddenly increased several times, and the validation accuracy in the right graph suddenly decreased several times. Analysis

The results show that both weight decay coefficients performed well on the training set, ultimately improving training accuracy to over 99% and reducing training loss to very low levels. Specifically, λ=5×10−4\lambda=5\times10^{-4}λ=5×10−4 yielded the following results: training loss 0.0125, training accuracy 99.85%, validation loss 0.9589, and validation accuracy 78.48%, while λ=1×10−4\lambda=1\times10^{-4}λ=1×10−4 yielded the following results: training loss 0.0147, training accuracy 99.54%, validation loss 1.7228, and validation accuracy 71.05%. Although the difference between the two on the training set was small, the difference on the validation set was significant. λ=5×10−4\lambda=5 \times 10^{-4}λ=5×10−4 significantly outperformed in both validation loss and validation accuracy.

This result demonstrates that, under the current task and training settings, stronger regularization is more beneficial for generalization. The weaker constraint of λ=1×10−4\lambda=1\times10^{-4}λ=1×10−4 on the weights, though allowing the model to fit the training set well, doesn’t adequately suppress parameter size and lead to a tendency for the model to more closely fit the details in the training set, resulting in poor performance on the validation set. Conversely, λ=5×10−4\lambda=5\times10^{-4}λ=5×10−4 provides stronger weight constraints, preventing the model from over-relying on certain excessively large parameters during learning, thus reducing the risk of overfitting and ultimately achieving lower loss and higher accuracy on the validation set.

Data Augmentation

Introduction

This section adds a mixup to the existing setup, setting α=0.2 and training for 300 epochs. Mixup is a data augmentation method that generates new training samples by linearly interpolating two input images and their labels. It can improve sample diversity and improve the model’s generalization ability. Its basic form is to linearly interpolate two images and their corresponding labels as Formula 4:

x~=λxi+(1−λ)xj\tilde{x} = \lambda x_i + (1-\lambda)x_j x~=λxi​+(1−λ)xj​

y~=λyi+(1−λ)yj\tilde{y} = \lambda y_i + (1-\lambda)y_j y~​=λyi​+(1−λ)yj​

Where λ∼Beta(α,α)\lambda \sim \mathrm{Beta}(\alpha,\alpha)λ∼Beta(α,α) and when α=0.2\alpha=0.2α=0.2, Beta is more likely to be distributed around 0 and 1. Many mixed samples are still closer to one of the original samples, but some samples will more obviously mix the information of the two classes. Figure 3 shows the corresponding distribution.

The curve is high near 0 and 1, and low in the middle, around 0.5. That is when α=0.2\alpha=0.2α=0.2, λ\lambdaλ is more likely to approach 0 or 1, and less likely to fall in the middle.

Results

The comparison between the baseline and mixup is summarized in Table 5 and Figure 4 shows the corresponding training curves.

Table 5: Comparison between the baseline and mixup training.

Setting Train Loss Train Acc. Val Loss Val Acc.
No Mixup (baseline) 0.0129 99.80% 1.0964 76.03%
With Mixup (α=0.2\alpha = 0.2α=0.2) 0.4634 89.02% 1.1720 76.38%

The blue solid line (No Mixup training) indicates that the training accuracy eventually approaches 100%, while the green solid line (With Mixup training) shows a significantly lower training accuracy, hovering around 89%. The loss graph also reflects this, with the training loss for mixup training being significantly higher than the baseline. The two validation accuracy curves are very close, with the final validation accuracy for mixup training being slightly higher.

Analysis

The experimental results show that the baseline without mixup achieved the following in the last epoch: training loss 0.0129, training accuracy 99.80%, validation loss 1.0964, and validation accuracy 76.03%. With mixup, the model achieved the following results in the last epoch: training loss 0.4634, training accuracy 89.02%, validation loss 1.1720, and validation accuracy 76.38%. Furthermore, the mixup model used for final evaluation on the test set achieved a test accuracy of 79.18%.

The results indicate that mixup slightly worsens training set performance, but may provide more robust validation and test set performance. Adding mixup reduced training accuracy from nearly 100% to around 89%, and the training loss was significantly higher. Mixup itself makes the training task more difficult, actively suppressing the model’s over-memorization of the training set.

However, from a generalization perspective, the effect of mixup is positive. Although its final validation loss is slightly higher than the baseline, and the validation accuracy is only slightly higher (76.38% vs. 76.03%), the improvement is not significant. However, this indicates that despite a significant decrease in training set fitting, validation set performance did not decline; in fact, it slightly improved. This suggests that the mixup did indeed play a regularization role to some extent, sacrificing some training set fitting ability in exchange for slightly better generalization performance.

With the α=0.2 setting, since λ∼Beta(0.2,0.2), the mixing coefficient will mostly be closer to 0 or 1. Therefore, many generated samples still resemble one of the original categories, only with a small amount of information from another category added. This mixup strength is relatively mild, preventing the training samples from becoming overly distorted, thus allowing the model to maintain good learning performance. At the same time, it is sufficient to increase the diversity of training samples, mitigating the risk of overfitting. Therefore, while the improvement is not particularly large, the direction is reasonable.
